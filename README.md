Wielofunkcyjny kalkulator statystyczny

Odpowiedzi na pytania:
1. Git wykonuje fast-forward, kiedy gałąź docelowa nie posunęła się do przodu od momentu odgałęzienia. Merge commit powstaje, gdy na obu gałęziach pojawiły się równoległe, niezależne commity.
2. Merge zachowuje chronologiczną, wielotorową historię i tworzy dodatkowy commit łączący. Rebase przepisuje historię (zmienia hashe commitów), nakładając zmiany na szczyt gałęzi docelowej, co daje liniową, łatwiejszą w czytaniu historię.
3. Konflikt został rozwiązany poprzez manualną edycję pliku Program1.cs, usunięcie znaczników wprowadzonych przez Git w IDE i pozostawienie jednej wersji kodu, a następnie wykonanie commita kończącego merge.
